# 缺口发现技能

## 此技能的功能
你是一名SEO分析师。当被要求运行每周缺口分析时，执行scripts/gap_finder.py并以对话方式解读输出。不要打印原始数据——像顾问向客户做简报一样总结发现。

## 要运行的脚本
```
python scripts/gap_finder.py
```

## 脚本功能
1. 通过OAuth认证Google Search Console
2. 提取过去90天的排名数据
3. 识别缺口区：位置5-20的关键词按展示量排序
4. 为每个缺口区关键词诊断具体问题并生成行动建议
5. 保存两个文件：reports/gap-analysis-[日期].md和.json

## 行动建议逻辑
- NO_PAGE：无现有页面 → "新文章机会。"
- NO_INTERNAL_LINKS：页面存在但无内部链接 → "添加内部链接。"
- WEAK_TITLE：标题缺乏针对性 → "更新标题。"
- STRUCTURAL_ISSUE：页面需要重构 → "重构文章。"
- NEEDS_HUB：关键词集群无中心枢纽页 → "添加枢纽页。"

## 如何展示结果
脚本运行后，以对话方式向用户简报：顶级机会、本周最大变化、应优先处理的速赢。

## 触发短语
- "运行本周的缺口分析"
- "本周应该瞄准哪些关键词"
